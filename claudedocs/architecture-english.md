# AI 튜터 (가칭) — 영어 과목 아키텍처 (확장 설계)

> 작성일: 2026-06-22
> 상태: 초안 (Phase 3 확장 설계)
> 상위 문서: [architecture.md](architecture.md) — 인증/역할/RLS/이미지/가드레일 등 공통 인프라는 메인 문서 참조. 본 문서는 **영어 과목 고유 사항만** 다룬다.

---

## 1. 왜 별도 설계가 필요한가

영어는 수학과 학습 구조가 근본적으로 다르다.

| | 수학 | 영어 |
|---|---|---|
| 정답 형태 | 단일 수치/식 (객관적) | 복수 정답 가능 (해석·표현) |
| "답 누설" 정의 | 최종 답·완성 풀이 | **정답 선택지 / 통째 번역 / 빈칸 정답어** |
| 핵심 매체 | 수식·도형 렌더 | 본문 텍스트·음성(듣기) |
| 약점 태깅 | 단원·유형 | 어휘·문법·구문·독해기능 |

→ 공통 파이프라인(입력 가드 → 튜터 → 출력 검증)은 재사용하되, **"답 누설"의 정의와 검증 규칙, 콘텐츠 타입**이 달라진다.

---

## 2. 영어 학습 영역(서브도메인)

MVP 이후 확장 시 우선순위:
1. **독해(Reading)** — 지문 + 문항. 가장 수요 크고 사진 입력과 잘 맞음.
2. **문법(Grammar)** — 어법 문항, 오류 찾기.
3. **어휘(Vocabulary)** — 문맥 추론, 누적 단어장.
4. **듣기(Listening)** — 음성 생성(TTS) 기반. 기술 난이도 가장 높음 → 마지막.

---

## 3. "답을 안 알려준다" — 영어판 가드레일

수학의 solve/review 모드를 계승하되, **영어 특유의 누설 경로**를 추가 차단한다.

### 3.1 차단해야 할 누설 경로
- ❌ 객관식 **정답 번호** 직접 제시
- ❌ 빈칸/어법 문제의 **정답 단어·형태** 직접 제시
- ❌ 지문 **전체 번역** 제공 (해석을 대신 해줘버림 = 학습 무력화)
- ❌ 영작·서술형의 **모범답안 통째 제시**

### 3.2 대신 제공하는 것 (소크라테스식)
- ✅ 모르는 **단어의 뜻·품사 힌트** (단, 정답을 직접 결정짓는 핵심어는 단계적으로)
- ✅ 문장 **구조 분해**(주어/동사/수식 관계) — 번역은 학생이 직접
- ✅ 문항이 **무엇을 묻는지** 재진술, 함정 포인트 안내
- ✅ 오답 선택지가 **왜 매력적 오답인지**의 사고 과정

### 3.3 모드 + 이해도 게이트 (메인 §5와 동일 구조)
- **이해도 게이팅 계승**: 영어도 "절대 안 줌"이 아니라 이해도 ≥90%에서 답 공개. 단 "이해도"의 근거가 영어식이다 — 학생이 **자기 말로** 지문/문장을 해석하고, 정답 근거와 오답 배제 이유를 시연했는가. "정답 1번이지?" 추측엔 동일하게 "왜 그렇게 생각해?"로 근거 요구.
- **solve 모드**: 미공개 문제 → 90% 전엔 정답·전체 번역·모범답안 금지, 힌트/구조분해만.
- **review 모드**: 답·해설 있는 문제 → "왜 이 선택지가 답인지", "이 문장 구조 왜 이렇게 해석되는지" 해설 허용.

### 3.4 출력 검증 레이어 추가 규칙
- 응답에 **지문 문장의 연속 번역이 일정 길이 이상** 포함되면 → redact.
- 객관식 정답 번호/빈칸 정답어가 직접 노출되면 → redact.
- 학년(`grade`) 기반 어휘·문법 범위 스코핑(과메타적 설명 방지).

## 4. 듣기(Listening) 생성 파이프라인 — Phase 3 후반

> 기술 난이도·비용이 가장 높은 영역. 별도 일정으로 분리.

```
스크립트 생성(LLM, 학년·주제·난이도 지정)
  → 화자 분리·발화 단위 분해
  → TTS 합성(다화자/속도 조절) → 오디오 파일(R2/Storage)
  → 문항 생성(내용 일치/추론 등) + 가드레일(정답 비노출)
  → 학생: 오디오 재생 + 문항 풀이, AI는 힌트만
```
- TTS: 자연스러운 다화자·억양 지원 엔진 필요(후보 비교 별도 검토).
- 비용 관리: 생성 오디오 **캐싱·재사용**(같은 스크립트 재합성 금지).
- 저작권: 실제 시험 음원 사용 금지, **생성 콘텐츠만**.

## 5. 데이터 모델 확장 (메인 §4에 더하여)

```
conversations.subject        -- enum 확장: math | english
conversations.english_domain -- enum: reading | grammar | vocab | listening (nullable)

passages   (독해 지문 — 재사용·검색 대상)
  id, source_image_url, body_text, grade_band, topic_tag, created_at

listening_assets  (Phase 3)
  id, script_text, audio_url, voices(json), duration_sec, created_at

-- 어휘 누적(영어판 오답노트)
vocab_items
  id, student_id, term, context_sentence, mastery_level, last_seen_at
```
- 수학의 `mistakes`와 영어의 `vocab_items`는 별개 누적 테이블. Phase 2 모의고사 엔진이 둘 다 소비.

## 6. 푸시 알림 (공통 모듈 — 메인 문서에도 반영)

> DB 선택과 무관. **Web Push(VAPID) 표준** 사용, Firebase 불필요.

```
서비스워커 등록 → 푸시 구독(PushSubscription) → Supabase에 저장
  → 서버(Edge Function)에서 web-push(VAPID)로 발송
```
- `push_subscriptions` 테이블: `student_id`, `endpoint`, `keys(p256dh,auth)`, `created_at`.
- ⚠️ **iOS는 "홈 화면에 추가"된 PWA standalone 상태에서만** 푸시 수신(Safari 16.4+). 브라우저 탭은 미지원.
- 발송 트리거 예: 선생님이 과제 등록, 모의고사 준비 완료, 복습 리마인더(에빙하우스 주기).
